
# F1 Lap Time & Race Strategy Predictor

This project utilizes machine learning (XGBoost) and the [FastF1](https://docs.fastf1.dev/) API to analyse and predict Formula 1 lap times. By extracting real race telemetry, weather conditions, tyre life and track position data. It then trains a model that understands how the selected data can affect tyre performence and lap times. 

*Note: The pit stop prediction functionality is actively being developed and is currently a work in progress.*

## Installation Instructions

## Requirements

Install the required dependencies using pip:

```bash
pip install -r requirements.txt
```

## Project Structure
- `dataset.py`: This script is used to fetch and process the selected F1 telemetry data using the FastF1 API. This creates the parquet file which the model uses to create results.
- `model.py`: This script loads the processed dataset and trains an XGBoost regression model to predict lap times. It then evaluates the model's performance using metrics like MAE, RMSE, and R-squared.
- `f12025Data.parquet`: This is the output dataset generated by `dataset.py`, containing the processed telemetry data for the 2025 F1 season. It is used as input for `model.py` to train and evaluate the predictive model.

## Dataset Information

The dataset generation relies on the [FastF1 API](https://docs.fastf1.dev/). This automatically fetches real telemetry such as lap times, driver positions and weather data. Varibles like track length, tire compounds, and weather conditions are retrieved directly from the API and do not require manual configuration.

### Parameters to Adjust

To change which races or sessions are included in your dataset, you can modify the `fastf1.get_session()` parameters directly inside `dataset.py`:

- **`year`**: The championship year (e.g., `2025`).
- **`round`**: The round number of the race calendar. The script currently loops through rounds using `for round in range(1, 25):`. To fetch a specific race, just provide its corresponding round number or partial event name such as `Australia`.
- Session Type: The session type. Default is `'R'` for Race, but it can be changed to `'Q'` (Qualifying), `'SQ'` (Sprint Shootout), `'S'` (Sprint), or `'FP1'`/`'FP2'`/`'FP3'` (Practice sessions).

### Example Configuration

If you only want to extract data for a single specific event. For example the 2025 Australian Grand Prix (Round 1)/ The `raceSession` variable in `dataset.py` can be modified to target just that specific session rather than looping:
#### Option 1: Using Round Number
```python
# Load session and lap data for a specific race (2025, Round 1, Race)
raceSession = fastf1.get_session(2025, 1, 'R')
raceSession.load()
```
#### Option 2: Using Partial Event Name
```python
# Load session and lap data for a specific race (2025, Australian Grand Prix, Race)
raceSession = fastf1.get_session(2025, 'Australia', 'R')
raceSession.load()
```

## Model Output Results

When running `model.py`, the script evaluates the XGBoost model's performance on the test dataset and outputs metrics like the ones below:

```text
Mean Absolute Error (MAE): 0.358 seconds
Root Mean Squared Error (RMSE): 0.536 seconds
R-squared (R2): 0.997
```

### Understanding The Metrics

- **Mean Absolute Error (MAE)**: This tells you how far off the models lap time predictions are on average. In the results above, the model's predictions are **0.358 seconds** away from the actual lap time, on average. The lower the MAE, the more accurate the models predictions are.
- **Root Mean Squared Error (RMSE)**: The RMSE is similar to MAE but gives a higher penalty to larger prediction errors. An RMSE of **0.536 seconds** means the standard deviation of the prediction errors is around half a second. The lower the RMSE, the more accurate the models predictions are.
- **R-squared (R2)**: This shows how well the model matches the real data. A score of **0.997** means the model successfully captures 99.7% of the patterns affecting lap times. A perfect score is 1.0, so the closer to 1.0, the more accurate the model is.
